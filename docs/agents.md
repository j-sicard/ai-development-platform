# Les Agents IA

## Présentation

AI Studio repose sur une équipe de neuf agents IA spécialisés.

Chaque agent représente un rôle expert d'une équipe de développement logiciel traditionnelle.

Les agents possèdent :

* un rôle clairement défini ;
* un domaine d'expertise ;
* des connaissances associées ;
* des responsabilités spécifiques dans le cycle de vie d'un projet.

Ils collaborent afin de transformer un besoin utilisateur en une solution logicielle structurée.

---

# Organisation de l'équipe

L'équipe d'agents couvre les différentes phases d'un projet logiciel :

```text
Besoin utilisateur

        |

        v

Product Owner
        |
        v

Research Analyst

        |
        v

Architecte Logiciel

        |
        +----------------+
        |                |
        v                v

Backend Developer   Frontend Developer

        |
        v

QA Engineer + Security Engineer

        |
        v

DevOps Engineer

        |
        v

Technical Writer
```

---

# 1. Product Owner / Business Analyst Senior

## Rôle

Analyser les besoins métier et transformer une idée utilisateur en fonctionnalités clairement définies.

## Responsabilités

* comprendre les objectifs du projet ;
* identifier les utilisateurs ;
* rédiger les besoins fonctionnels ;
* créer les user stories ;
* définir les priorités ;
* valider la cohérence fonctionnelle.

## Connaissances associées

* Méthodologie Projet Logiciel ;
* Analyse Métier et Besoins.

---

# 2. Research Analyst Senior

## Rôle

Effectuer des recherches et analyser les solutions existantes afin d'aider aux choix techniques et fonctionnels.

## Responsabilités

* rechercher des informations techniques ;
* comparer des solutions ;
* analyser les technologies disponibles ;
* identifier les bonnes pratiques.

## Connaissances associées

* Méthodologie Projet Logiciel ;
* Analyse Métier et Besoins.

---

# 3. Architecte Logiciel Senior

## Rôle

Concevoir la structure technique globale d'une application.

## Responsabilités

* définir l'architecture ;
* choisir les composants ;
* proposer les technologies ;
* analyser les contraintes ;
* documenter les décisions techniques.

## Connaissances associées

* Méthodologie Projet Logiciel ;
* Architecture ;
* Bases de Données ;
* Sécurité Applicative.

---

---

# Collaboration entre agents

Les agents ne travaillent pas de manière isolée.

Ils partagent :

* les besoins du projet ;
* les décisions techniques ;
* les documents produits ;
* la mémoire du projet.

Chaque agent contribue selon son domaine d'expertise afin de construire une solution cohérente.

---

# Évolution future

L'objectif d'AI Studio est de permettre aux agents de collaborer automatiquement grâce à :

* une mémoire projet ;
* une architecture RAG ;
* une base vectorielle ;
* une orchestration des workflows.
